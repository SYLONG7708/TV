自動更新方式

YouTube 真實播放 URL 是短效網址，不能永久固定。本 repo 已加入 GitHub Actions，每 2 小時自動執行 tools/update-youtube-live.ps1，用 yt-dlp 重新擷取公開直播 URL。

腳本會優先選擇 720p HLS，降低手機、電視盒與模擬器長時間播放時的卡頓機率，並合併到 APK 已經使用的 sources/live-stable.txt。

解析報告：sources/live-youtube-report.json

APK 直播 URL 不變：https://raw.githubusercontent.com/SYLONG7708/TV/main/sources/live-stable.txt

手動立即更新

powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\update-youtube-live.ps1 -DownloadYtDlp -IncludeOriginalOnFailure

新增或調整頻道

用文字編輯器打開 sources/youtube-live-channels.csv，修改 Order、Group、Name、Url。儲存後執行上方指令，或等待 GitHub Actions 自動刷新。

地區限制、影片下架、非公開或 DRM 內容無法被腳本強制播放；腳本會保留原 YouTube 頁面 URL，並把失敗原因寫入 sources/live-youtube-report.json。

沒 cookies 的 100% 成功模式

GitHub runner 沒有 YouTube cookies 時，workflow 會使用 no-cookies-fallback 模式，保留 98 個原始 YouTube 直播 URL。這樣 Actions 會 100% 成功更新清單，不會卡在 YouTube 機器人驗證；報告中的 workflowSuccessRate 會是 100。

這代表清單更新成功，不代表已取得短效 HLS。若要提高 hlsSuccessRate，仍需設定 YOUTUBE_COOKIES_B64。

GitHub 被 YouTube 擋住時

如果報告檔出現 Sign in to confirm you're not a bot，代表 GitHub runner 被 YouTube 要求登入。先在已登入 YouTube 的電腦產生 cookies，再把 Base64 放到 GitHub Secret。

.\.tools\yt-dlp.exe --cookies-from-browser chrome --cookies youtube-cookies.txt --skip-download "https://www.youtube.com/"
[Convert]::ToBase64String([IO.File]::ReadAllBytes(".\youtube-cookies.txt")) | Set-Clipboard

到 GitHub repo 的 Settings → Secrets and variables → Actions → New repository secret，名稱填 YOUTUBE_COOKIES_B64，內容貼上剪貼簿的 Base64 字串。cookies 是登入憑證，只能放在 GitHub Secret，不要上傳到 repo。

powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\update-oktv-sources.ps1 `
  -InputApk .\releases\OKTV_5.1.6_builtin_sources.apk `
  -OutputApk .\releases\OKTV_5.1.6_custom_sources.apk `
  -VodUrl "你的點播 JSON URL" `
  -LiveUrl "你的直播 TXT/M3U URL"

要重新抓取原始直播源、去重、每條線路短測 2 次並排序備援線路，執行：

        
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\build-stable-live.ps1

        
完成後會更新 sources/live-stable.txt，短測通過的線路會出現在 Verified Fastest 優先分類，私密頻道會保留密碼 7708，APK 目前已內建讀取這個穩定版 URL。

常見問題

        
安裝時提示簽名不同

        
先卸載已安裝的原版，再安裝這個重新簽名版本。

        
PowerShell 說找不到 java

        
安裝 JDK，並確認重新開啟 PowerShell 後輸入 java -version 能看到版本。

        
PowerShell 說找不到 Android build-tools

        
安裝 Android SDK build-tools，或執行腳本時加上 -AndroidSdk "你的 Android SDK 路徑"。

        
來源不能播放

        
先用瀏覽器打開 URL，確認檔案存在、網路可連線，而且來源格式符合 app 支援格式。

        
請避免使用沒有授權的第三方內容來源。來源能否播放與內容授權狀態需要由來源維護者自行確認。
